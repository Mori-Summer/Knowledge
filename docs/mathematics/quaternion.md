# 四元数：一种带非交换乘法的四维实代数对象

## 1. 这份文档要帮你学会什么

这篇文档的重点，不是把四元数继续写成一篇“3D 旋转实战指南”，而是先把四元数这个概念本体讲清。

读完后，你应该至少能做到：

- 说清四元数为什么不是“4 维向量换个名字”，而是带非交换乘法的对象
- 区分四元数本体、纯四元数、单位四元数和“用四元数表示旋转”这几个层次
- 知道什么时候在讨论四元数代数本身，什么时候已经切到姿态表示与工程应用
- 不再把“4 个数存旋转”当成四元数的全部定义
- 知道这篇文档停在概念边界，应用、比较和选型要去看下游文档

## 2. 一句话结论 / 概念结论

**四元数是形如 `q = w + xi + yj + zk` 的对象，其中 `w`、`x`、`y`、`z` 是实数，`i`、`j`、`k` 满足特定乘法规则；它首先命名的是一个带非交换乘法的四维实代数，而不是“任何四参数旋转表示”的统称。**

如果只记一句最稳的话：

**四元数首先解决的不是“3D 旋转怎么存”，而是“我们需要一个怎样的完整对象，既能保留加法结构，又能让乘法携带方向与组合次序信息”。**

## 3. 这个概念试图解决什么命名或分类问题

一旦从复数再往上走，最容易混在一起的是下面几类对象：

- 四元数本体 `w + xi + yj + zk`
- 纯四元数 `(0, x, y, z)`
- 单位四元数
- 任意四维实向量
- 旋转矩阵、欧拉角、轴角等 3D 旋转表示

如果没有“四元数”这个概念本体，后面很容易出现几类错位：

- 把“单位四元数表示旋转”误当成“四元数的全部定义”
- 把“4 个数的存储格式”误当成“四元数”
- 把四元数和普通 4D 向量混成一类
- 把四元数和旋转矩阵、欧拉角看成只是“不同 API 格式”

所以，“四元数”这个概念首先在做的不是选型，而是分层：

- 给这类代数对象本体一个稳定名字
- 把对象本体和其中特殊子集区分开
- 把数学对象和下游旋转用途分开

## 4. 概念边界与相邻概念

### 4.1 这篇文档直接处理什么

这篇文档直接处理的是：

- 四元数 `q = w + xi + yj + zk`
- 标量部与向量部
- 共轭、范数、逆
- 单位四元数作为一个特殊子集

这篇文档不负责系统展开：

- 四元数在 3D 旋转里的具体工作流
- 四元数和欧拉角、旋转矩阵、旋转向量的工程比较
- 姿态积分、插值、误差建模和约定选择

### 4.2 它不等于什么

四元数不等于：

- **普通 4 维向量。**
  普通向量没有四元数那套乘法规则。

- **所有四参数姿态表示。**
  只有四个数，不等于就是四元数。

- **单位四元数本身。**
  单位四元数只是四元数里的一个特殊子集。

- **旋转矩阵的另一种写法。**
  它们可以描述相关几何内容，但对象类型和内部约束不同。

- **欧拉角的升级版。**
  欧拉角是坐标参数化；四元数是另一类代数对象。

### 4.3 最容易混淆的相邻对象

| 对象 | 它是什么 | 和四元数的关系 |
| --- | --- | --- |
| 复数 | 带交换乘法的二维实代数对象 | 可视为更低维的相关代数对象，但不是四元数 |
| `(w, x, y, z)` | 四个实数构成的有序组 | 若不带四元数乘法语义，它还不是四元数本体 |
| 纯四元数 | 标量部为 `0` 的四元数 | 是四元数中的特殊子类 |
| 单位四元数 | 范数为 `1` 的四元数 | 是与 3D 旋转最相关的特殊子集 |
| 旋转矩阵 | 作用于 3D 向量的线性变换表示 | 和单位四元数可互相转换，但不是同一种对象 |
| 欧拉角 | 某种局部坐标参数化 | 不是四元数，也不是四元数的子类 |

### 4.4 最关键的边界句

**看到“四元数”时，先追问你是在说四元数代数本体，还是在说其中的单位四元数子集；这两层不分开，后面的旋转讨论几乎一定会漂。**

## 5. 什么算它，什么不算它

下面这张表是最实用的判别模板。

| 表达或对象 | 在本文里怎么处理 | 为什么 |
| --- | --- | --- |
| `1 + 2i - j + 3k` | 是四元数 | 它满足四元数对象形式 |
| `(0, x, y, z)` | 是纯四元数 | 它是四元数的一个子类 |
| `\cos(\theta / 2) + \mathbf{u}\sin(\theta / 2)` 且范数为 `1` | 是单位四元数 | 它落在特殊子集里 |
| 四个浮点数组成的普通数据结构 | 不一定算四元数 | 只有当它承载四元数运算语义时才是 |
| `3 x 3` 旋转矩阵 | 不是四元数 | 它是另一种表示对象 |
| 欧拉角 `(roll, pitch, yaw)` | 不是四元数 | 这是参数化，不是四元数本体 |
| 双四元数 | 不是普通四元数 | 它是另一类更大的对象 |

最值得反复提醒的一点是：

**不是所有四元数都表示旋转；只有单位四元数才直接进入纯旋转语义。**

## 6. 代表性例子、反例与常见误读

### 6.1 代表性例子

最有代表性的例子有四个：

- `i`、`j`、`k`
  这提醒你：四元数不是“复数多加两列”，而是引入了新的乘法方向。

- `1 + i`
  这是一个最简单的非纯四元数例子，说明四元数首先是完整对象，而不只是旋转格式。

- `(0, x, y, z)`
  这提醒你：纯四元数是特殊子类，不是整个四元数本体。

- 单位四元数 `q = \cos(\theta / 2) + \mathbf{u}\sin(\theta / 2)`
  这提醒你：和旋转最相关的是单位四元数子集，而不是任意四元数。

### 6.2 反例

最值得记住的反例有下面这些：

- 一个随机四元组 `(0.1, 2.0, -3.5, 7.2)`
  如果你只把它当四列数值，没有四元数语义，它还不是概念本体。

- 一个 `3 x 3` 旋转矩阵
  它能表示旋转，但它不是四元数。

- 欧拉角 `yaw/pitch/roll`
  这不是四元数，只是另一类参数化方式。

- 把 `q` 和 `-q` 当成两种不同旋转
  这在旋转语义里通常是错的，因为它们对应同一几何旋转。

### 6.3 常见误读

最常见的误读有下面几类：

- **四元数就是旋转。**
  不稳。更准确的是：单位四元数常用来表示旋转。

- **四元数就是 4 维向量。**
  错。关键不在维数，而在乘法结构。

- **只要有四个参数，就等于在用四元数。**
  错。数据形状不等于对象类型。

- **用了四元数就自动没有约定问题。**
  错。主动/被动、左乘/右乘、分量顺序等问题仍然存在，但那属于下游应用层。

- **四元数乘法顺序只是实现细节。**
  错。非交换性是对象本体的一部分，不是注释问题。
